# pmgr — browser extension (M3.1 skeleton)

Manifest V3 extension. Loads the `hekate-core` WASM module and runs the
full single-user login + vault-list + copy-password flow inside the
toolbar popup.

## Build

From the workspace root:

```bash
make extension
```

This builds `hekate-core` for `wasm32-unknown-unknown`, runs
`wasm-bindgen`, and copies the resulting JS/WASM/TS files into
`clients/extension/wasm/`.

## Load it

1. Visit `chrome://extensions` (or `edge://extensions`,
   `about:debugging` on Firefox — Firefox uses MV3 with minor
   differences but this skeleton targets Chromium).
2. Toggle **Developer mode** on.
3. Click **Load unpacked** and pick the `clients/extension`
   directory.
4. Click the pmgr icon in the toolbar to open the popup.

## What works in M3.1

- Login form: server URL + email + master password
- Master-key derivation client-side (Argon2id via WASM)
- OAuth password grant against a hekate server
- Account-key unwrap client-side
- Vault list with decrypted names + username/uri preview
- Per-row "Copy" button → password copied to clipboard
- "Lock" button: clears `chrome.storage.session`

## What's not in M3.1 yet

- Autofill via content script (M3.2)
- Add / edit / delete from the popup (M3.2+)
- Auto-refresh on 401
- TOTP code display
- Per-cipher-type rendering for cards / identities / SSH keys
- Browser-stored remember-me / device approval

## Layout

```
clients/extension/
├── manifest.json          MV3 manifest
├── popup/
│   ├── popup.html         shell
│   ├── popup.css          minimal light/dark UI
│   └── popup.js           ES module — drives the entire flow
└── wasm/                  generated by `make extension`
    ├── hekate_core_bg.wasm  ~195 KB
    ├── hekate_core.js       ~30 KB JS glue
    └── hekate_core.d.ts     TS definitions
```

## Notes

- Session state lives only in `chrome.storage.session` (RAM-only,
  cleared on browser close). Master password and master key are
  never persisted; the account key is held in session for the life
  of the popup-open + browser-open window.
- Last-used server URL and email are remembered in `localStorage`
  (non-secret) so you don't retype them.
- CSP is `script-src 'self' 'wasm-unsafe-eval'` — required for
  WebAssembly compilation in MV3 extension pages.
